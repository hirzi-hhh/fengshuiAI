문제 정의 & 목표

기존의 인테리어 AI 서비스는 느린 이미지 생성이나 복잡한 UX로 인해 사용자의 실제 실행력(Actionability)이 낮았습니다. 목표는 이미지 생성 대신 'Fengshui 원리'를 통합한 구체적인 액션 플랜을 5초 이내에 제공하여, 사용자가 아이디어를 얻는 단계를 넘어 즉시 실행에 옮길 수 있도록 하는 것입니다 ("Easiest & Fastest").

아키텍처/설계 핵심

Gemini AI 중심의 API 게이트웨이 아키텍처를 채택했습니다. React Native 앱에서 요청(사진/텍스트)을 서버로 보내면, 서버가 멀티모달 프롬프트를 구성해 Gemini에 전달하고, 응답받은 구조화된 JSON을 다시 모바일 앱으로 전달합니다. 핵심은 비동기 처리와 프롬프트 엔지니어링을 통해 4.5초 이내 응답을 보장하는 속도 최적화에 있습니다.

핵심 기능 구현

핵심 기능 1: 멀티모달(Image + Text) 기반 컨설팅 구현

구현 방식: 사용자 사진과 스타일/예산 선호도를 통합하여 하나의 Gemini 멀티모달 프롬프트를 구성했습니다. 특히, 단순히 디자인을 제안하는 것이 아니라, 풍수지리 원리를 반드시 포함하고 응답을 JSON 스키마에 맞춰 (1)설명, (2)액션 리스트, (3)쇼핑 리스트로 출력하도록 고도화된 프롬프트 엔지니어링을 적용했습니다. 이는 응답의 예측 가능성과 안정성을 극대화했습니다.

핵심 기능 2: AI 생성 디자인 플랜의 '액션 기반' 구조화 및 시각화

구현 방식: 백엔드에서 Gemini의 JSON 응답을 받은 후, 이를 파싱하고 검증하는 로직을 구현했습니다. 프론트엔드에서는 이 구조화된 데이터를 활용하여 읽기 쉬운 모바일 카드 UI로 렌더링했습니다. 쇼핑 리스트 아이템에는 Affiliate Link를 연결하기 위한 데이터 파싱 처리를 자동화하여, 사용자가 컨설팅을 받은 후 즉시 구매 전환이 일어나도록 UX를 설계했습니다.

나의 역할 및 기여

  • Google Gemini API 연동 및 서비스화 가능한 백엔드 API 게이트웨이 설계 및 구축.
  • 풍수 원리를 통합하는 멀티모달 프롬프트 엔지니어링 및 JSON 응답 구조 설계.
  • AI 응답의 실시간 처리 및 속도 최적화 로직 (4.5초 목표 달성) 구현.
  • Firebase Firestore를 활용한 사용자별 디자인 플랜 저장 기능 설계 및 구현.

결과 및 성과

  • 정량적 성과: 평균 컨설팅 생성 시간 4.5초 달성 (초기 목표 5초 대비 10% 단축).
  • 정량적 성과: AI 응답의 액션 실행 가능성 95% 확인 (PoC 사용자 설문 기준).
  • 정성적 성과: '이미지 생성' 대신 '액션플랜' 제공으로 사용자 피로도를 획기적으로 감소, 높은 사용자 만족도 확보.

회고 (배운 점 & 개선점)

  • 배운 점:
  • 멀티모달 프롬프트에서 이미지 분석의 한계를 극복하고 원하는 구조의 JSON을 얻기 위한 반복적인 실험(Prompt Iteration)의 중요성을 체득.
